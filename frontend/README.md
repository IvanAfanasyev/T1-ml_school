# Frontend

`frontend/` - статический интерфейс CloudMatch. Он не требует сборщика, Node.js или отдельного frontend-сервера. Достаточно отдать `index.html`, `styles.css` и `app.js` через браузер или Nginx.

## Структура

```text
frontend/
├── README.md
├── index.html
├── styles.css
└── app.js
```

## Основные файлы

`frontend/index.html` содержит разметку сайта: верхнюю навигацию, вкладку витрины, вкладку агента, страницу о проекте и модальное окно карточки сервиса.

`frontend/styles.css` отвечает за визуальный стиль: сетку карточек, шапку, вкладки, чат, сообщения пользователя и агента, связки сервисов, адаптивность под мобильные экраны.

`frontend/app.js` содержит всю клиентскую логику: загрузку каталога, переключение вкладок, отправку сообщений агенту, сохранение истории чата, отрисовку рекомендаций и открытие карточек сервисов.

## Вкладки сайта

### Витрина

Витрина загружает сервисы через:

```text
GET /api/catalog/services
```

На карточке показываются:

- провайдер;
- название сервиса;
- описание;
- категория;
- регион;
- технология или тег;
- цена и месячная оценка для почасовых тарифов, если она есть в нормализованных данных;
- кнопка подробной карточки;
- ссылка на сайт сервиса.

Подробная карточка загружается через:

```text
GET /api/catalog/services/{service_id}
```

Там показываются тарифные позиции, которые связаны с сервисом через `service_id`.

### Агент

Вкладка агента отправляет сообщения в:

```text
POST /api/chat
```

Frontend передает backend:

- `user_id`;
- `chat_id`;
- текст сообщения;
- текущую `memory`;
- флаг `with_explanation`;
- флаг `include_debug`.

Backend возвращает:

- действие агента;
- текст ответа;
- вопросы для уточнения;
- результаты поиска;
- обновленную память.

### О проекте

Статическая вкладка с кратким описанием проекта и ссылкой на репозиторий.

## Память чата

История чата хранится в браузере в `localStorage`.

Ключ:

```text
cloudmatch.chat.v1
```

Что хранится:

- `chat_id`;
- список сообщений;
- последняя память backend.

Что это дает:

- backend не хранит личные сообщения пользователя;
- пользователь может продолжать один чат после перезагрузки страницы;
- кнопка `Сбросить чат` полностью очищает историю в текущем браузере.

Ограничение: если открыть сайт в другом браузере или на другом компьютере, старая история не появится.

## Поведение поля ввода

- `Enter` отправляет сообщение;
- `Shift + Enter` переносит строку;
- кнопка отправки блокируется на время запроса;
- при ошибке backend пользователь видит понятное сообщение.

## Как выбирается API

В `app.js` есть функция `getApiBase()`.

Правила:

- если сайт открыт локально, API = `http://127.0.0.1:8000`;
- если сайт открыт с сервера, API = текущий домен;
- если заранее задан `window.CLOUDMATCH_API_BASE`, используется он.

Это позволяет одним и тем же frontend-файлам работать локально и на сервере.

## Как frontend отображает ответы

Если backend просит уточнение, frontend показывает компактный блок с вопросами.

Если backend вернул обычный поиск, frontend показывает top-3 карточки провайдеров:

- место;
- провайдер и сервис;
- категория;
- цена;
- совпавшие параметры;
- что нужно проверить отдельно;
- объяснение;
- ссылка на сервис.

Если backend вернул комплексный запрос, frontend группирует сервисы по связкам:

```text
#1 Провайдер — сервис + Провайдер — сервис + ...
#2 Провайдер — сервис + Провайдер — сервис + ...
```

Внутри каждой связки показываются роли: backend, database, storage, backup, balancer и другие.

## Локальный запуск

1. Запустить backend:

```bash
uvicorn backend.app.main:app --host 127.0.0.1 --port 8000 --reload
```

2. Открыть файл:

```text
frontend/index.html
```

Если браузер блокирует запросы при открытии файла напрямую, можно поднять простой static server:

```bash
python3 -m http.server 5173 --directory frontend
```

После этого открыть:

[http://127.0.0.1:5173](http://127.0.0.1:5173)

## Production

В production frontend отдается Nginx как статические файлы.

Обычно это выглядит так:

```text
Nginx
  /             -> frontend/index.html
  /styles.css   -> frontend/styles.css
  /app.js       -> frontend/app.js
  /api/...      -> backend container
```
