# Тестирование

Базовая проверка проекта:

```bash
python -m unittest
node --check frontend/app.js
```

Что покрывают тесты:

- структуру нормализованных JSON;
- извлечение типа запроса и технологий;
- диалоговые уточнения;
- ответ `мне все равно` и `любой бюджет`;
- сброс старого контекста при новом запросе;
- региональные fallback-правила;
- ranking и score;
- комплексные связки сервисов от одного провайдера.

Ручные сценарии для проверки через `/docs` или frontend:

1. `база данных MySQL, москва, любой бюджет`
   - агент не должен спрашивать бюджет повторно;
   - первым сервисом должен быть MySQL.

2. `нужна база данных` -> `MySQL, москва, любой бюджет`
   - агент должен использовать MySQL, Москву и неважный бюджет;
   - старый PostgreSQL из прошлых сообщений не должен подтягиваться.

3. Большой запрос интернет-магазина:
   - backend на Python;
   - PostgreSQL;
   - хранение изображений;
   - backup;
   - масштабирование;
   - Россия;
   - бюджет до 50 000 руб./мес.

Ожидаемый тип ответа: связки сервисов. Внутри каждой связки все сервисы должны быть от одного провайдера.
