# Ограничения MVP

## Технические ограничения

- LLM используется для извлечения структуры запроса и объяснений, но не принимает финальное решение о ranking.
- История чата хранится во frontend через `localStorage`, не в БД.
- Нормализованные данные хранятся в JSON.
- Индексы нужно перестраивать после обновления данных.
- Полного калькулятора стоимости пока нет.

## Ограничения данных

- Используются только публичные источники.
- Не используются личные кабинеты и платные отчёты.
- Покрыты 4 провайдера.
- Обновление данных выполняется парсерами, не realtime.
- Качество рекомендаций зависит от полноты описаний в источниках.

## Бизнес-ограничения

- CloudMatch не является официальной интеграцией с провайдерами.
- Покупка услуги не выполняется внутри продукта.
- Рекомендации требуют проверки перед production.
- SLA, договоры и итоговые цены нужно подтверждать на стороне провайдера.

## Что пока не реализовано

- авторизация;
- личный кабинет;
- история чатов в БД;
- scheduled parsing;
- полноценный калькулятор конфигураций;
- мониторинг изменений тарифов;
- экспорт рекомендаций в PDF;
- user feedback loop.
